FIX: "Failed to Fetch" Error When Adding Members to Project
================================================================================

PROBLEM:
--------
When adding members to a project in Team Collaboration, you get "Failed to fetch" error.

ROOT CAUSE:
-----------
The @jwt_required(optional=True) decorator was making JWT optional, but the code
was trying to use get_jwt_identity() which returns None for OPTIONS requests,
causing the function to fail.

SOLUTION APPLIED:
-----------------
1. Removed @jwt_required(optional=True) decorator
2. Added manual JWT verification inside the function
3. OPTIONS requests bypass JWT check
4. POST/DELETE requests require valid JWT token
5. Added verify_jwt_in_request import

FILES CHANGED:
--------------
✅ backend/routes/team.py
   - Line 1133: Added verify_jwt_in_request import
   - Line 1309-1320: Fixed add_project_member function
   - Line 1351-1360: Fixed remove_project_member function

WHAT WAS FIXED:
---------------

BEFORE:
@team_bp.route('/projects/<project_id>/members', methods=['POST', 'OPTIONS'])
@jwt_required(optional=True)  ❌ JWT optional but code expects it
def add_project_member(project_id):
    if request.method == 'OPTIONS':
        return jsonify({'status': 'ok'}), 200
    
    try:
        current_user_id = get_jwt_identity()  ❌ Returns None for OPTIONS
        user = User.query.get(current_user_id)  ❌ Fails with None

AFTER:
@team_bp.route('/projects/<project_id>/members', methods=['POST', 'OPTIONS'])
def add_project_member(project_id):
    if request.method == 'OPTIONS':
        return jsonify({'status': 'ok'}), 200  ✅ Returns early
    
    try:
        verify_jwt_in_request()  ✅ Manually verify JWT for POST
        current_user_id = get_jwt_identity()  ✅ Now has valid token
        user = User.query.get(current_user_id)  ✅ Works correctly

EXPECTED BEHAVIOR:
