================================================================================
HOW TO TEST: Add Member to Project Feature
================================================================================

STEP 1: RESTART BACKEND
------------------------
1. Stop the backend if it's running (Ctrl+C)
2. Start it again:
   
   cd backend
   python run.py

3. You should see:
   * Running on http://127.0.0.1:5000
   * WARNING: This is a development server

STEP 2: OPEN YOUR APP
----------------------
1. Open browser: http://localhost:5173
2. Login with your account
3. Go to "Team Collaboration" page

STEP 3: CREATE OR SELECT PROJECT
---------------------------------
1. If you don't have a project:
   - Click "Create New Project" button
   - Enter project name and description
   - Click "Create Project"

2. If you have a project:
   - Click on the project card to open it

STEP 4: ADD A MEMBER
--------------------
1. Inside the project view, click "Add Members" button
2. You should see a modal with your collaborators list
3. Select a collaborator
4. Click the "Add" button next to their name

STEP 5: CHECK FOR SUCCESS
--------------------------
✅ You should see an alert: "✅ [email] added to project! They will see it in their Projects tab."
✅ The modal should close
✅ The member should appear in the project's members list

STEP 6: VERIFY IN BROWSER CONSOLE
----------------------------------
1. Press F12 to open Developer Tools
2. Go to "Console" tab
3. Try adding a member again
4. You should see:
   - API request to /api/team/projects/.../members
   - Response with success: true
   - NO red error messages

STEP 7: CHECK NETWORK TAB (OPTIONAL)
-------------------------------------
1. In Developer Tools, go to "Network" tab
2. Try adding a member
3. Look for the request: "members" (POST)
4. Click on it
5. Check:
   - Status: 200 OK
   - Response: {"success": true, "message": "..."}

================================================================================
WHAT TO LOOK FOR
================================================================================

✅ SUCCESS INDICATORS:
- Alert shows "✅ [email] added to project!"
- No errors in console
- Status 200 in Network tab
- Member appears in project

❌ ERROR INDICATORS:
- Alert shows "⚠️ Failed to add member"
- Red errors in console
- Status 401, 404, or 500 in Network tab
- CORS error messages

================================================================================
COMMON ERRORS & SOLUTIONS
================================================================================

ERROR: "Failed to fetch"
SOLUTION: 
- Check if backend is running (http://localhost:5000/api/health)
- Restart backend server

ERROR: "401 Unauthorized"
SOLUTION:
- Logout and login again
- Token might be expired

ERROR: "Project not found"
SOLUTION:
- Make sure you're the project owner
- Check if project ID is correct

ERROR: "Member already in project"
SOLUTION:
- This member is already added
- Try a different collaborator

ERROR: "CORS policy"
SOLUTION:
- Check backend/app.py CORS configuration
- Restart backend server

================================================================================
QUICK DEBUG COMMANDS
================================================================================

# Check backend health
curl http://localhost:5000/api/health

localStorage.getItem('token')

# Test add member manually (replace TOKEN and PROJECT_ID)
curl -X POST http://localhost:5000/api/team/projects/PROJECT_ID/members \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"email": "test@example.com"}'
